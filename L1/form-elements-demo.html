Basic Form Structure

The <form> element is the container for all form elements. It defines how data is submitted.

Form Attributes

  • action: Where to send form data
  • method: How to send data (GET/POST)
  • enctype: How to encode data

Common Input Types

HTML5 provides various input types for different kinds of data.

Text Inputs

  • type="text": Basic text input
  • type="email": Email address
  • type="password": Password field
  • type="tel": Telephone number

Special Inputs

  • type="date": Date picker
  • type="color": Color picker
  • type="file": File upload
  • type="range": Slider

Form Validation

HTML5 provides built-in form validation attributes.

Validation Attributes

  • required: Field must be filled
  • minlength: Minimum text length
  • maxlength: Maximum text length
  • pattern: Regular expression validation

Best Practices

Accessibility

  • Always use <label> elements
  • Group related inputs with <fieldset>
  • Provide clear error messages
  • Use appropriate ARIA attributes

User Experience

  • Use appropriate input types
  • Provide clear instructions
  • Show validation feedback
  • Make forms responsive

Pro Tip

Use the autocomplete attribute to help users fill out forms faster.

Security Note

Always validate form data on the server side, even if you have client-side validation.
