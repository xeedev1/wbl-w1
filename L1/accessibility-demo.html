Key Accessibility Attributes

HTML attributes play a crucial role in making web content accessible. Here are some essential attributes:

Basic Attributes

  • alt for images
  • title for tooltips
  • lang for language
  • tabindex for keyboard navigation

Form Attributes

  • required for mandatory fields
  • aria-describedby for descriptions
  • aria-label for labels
  • aria-invalid for validation

ARIA Roles and Landmarks

ARIA (Accessible Rich Internet Applications) roles help define the purpose of elements to assistive technologies.

Common ARIA Roles

  • role="alert" for important messages
  • role="banner" for site header
  • role="complementary" for sidebars
  • role="contentinfo" for footers
  • role="navigation" for navigation menus
  • role="search" for search forms

Accessibility Examples

Images

Good Example:

This image has a descriptive alt text that explains what the image shows and its context.

alt="A laptop displaying code on a dark screen, representing web development"

Bad Example:

This image has a non-descriptive alt text that doesn't provide any useful information.

alt="image"

Decorative Images

Good Example:

This decorative image has an empty alt text and role="presentation" to indicate it's purely decorative.

alt="" role="presentation"

Links

Good Example:

Learn more about our company

Bad Example:

Click here
